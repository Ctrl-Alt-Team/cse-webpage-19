What type of apps can I do?

Well, the goal is to be as creative and open as possible. In the first day we have a brainstorming session to discuss and choose ideas.

Remember: It has to involve Eurovision and Melodifestivalen.

But since we can't wait, we thought of some examples below:

  • Correlate data from previous competitions and find the probability of each country winning.
  • Identify any pattern (lyrics) among winners (or losers) of all the series.

Tips and tricks

Groups and teamwork

There is no I in TEAM. So, remember to build a good team with a variety of expertise so everyone can share and learn from one another.

Being friendly (not FRIENDS) is essencial for collaboration in a Hackathon. Especially among different teams. Remember, everyone is in the same boat: sink or swim.

You can come with a team of yours or meet people and join/form teams in our brainstorming evening. Remember that they have to be SEM students.

Delegating tasks is important. Explore everyone's unique capabilities: Divide and conquer!

Try to assign different responsibilities, for instance:

  • Coding: Front-end, back-end, ripping data from websites, ...

  • Presentation: Pitching the idea (slides, catch phrases), designing the interface, ...

  • Dev. Process: Using tools (git, APIs, etc.) properly, track time and goals, talk with organization,...

There are lots of tips on winning and enjoying Hackathon. Here is one, this is another, or you simply do this.

Above all: Be fun, creative and learn lots of new stuff (technical and personal).

Programming
languages and frameworks

Everyone feels that you need to know a lot of programming to participate in a hackathon. That is FALSE, or !true. Experience sure helps, but this is THE chance to learn and use new programming skills

About languages we suggest using the one you are more comfortable with, if none, then perhaps Python. Up to you, really.

Most of the programming will be script-based to analyse your dataset, so, for this Hackathon, don't worry too much about design.

Pick one and get going.

For different language search for frameworks of what you want to do. Usually web or mobile frameworks in Hackathons. Keep in mind that you should think of Web design, front-end and back-end development. Your database is the competition data (more here).

IDE and Tools

No matter the language you choose, having good development support goes a long way. So, get your team an effective way to:

  • Communicate: Talk with each other and other teams to share solutions and help each other;
  • Collaborate: Have version control (especially for code) so you can easily combine your solutions;
  • Code: Choose a good IDEs so you can code better and faster;

We are going to use Slack for communication (no, you should not use Facebook chat!). During the Hackathon you will send the invitations to the channels to communicate with your team, the organization and other teams.

You will use Github to share your project with your colleagues, so we suggest that you start now getting familiar with git.

Data scraping

Your application needs input, thus enter data. Remember, it has to be from Eurovision or Melodifestivalen.

There are lots of websites in the Internet containing crunched data and statistics, espcially for Eurovision.

The best one is here. But you can find data also here and there.

First step is to get that data into files (.csv, .txt, JSON, etc.) so you can use it as input for your program. You have two options:

  1. Extract the data manually;
  2. Study on how to use cool awesome libs to scrape the data off those webpages;

So, make sure you have a solution for getting the data. You can also come to the Hackathon with your dataset ready. Of course, we will check it if the data is okay. And we hope you can share it with other people too.

Some libs you can use to extract the data (search data scraping libs):

  • Mannually or semi-automatically: Webscraper;
  • Python: BeautifulSoup, see it working here;
  • Java: JSoup or Jaunt. Click here to see it in action.

Here is a nice and fun link to check options for Java and Python in Natural Language Processing (NLP) or Machine Learning (ML) . OpenNLP, is a lib for Java not included in that list.

Make sure you do some googling, currently there are so many options that its worthwhile to do some searching before choosing one (specially one with good tutorials).

Also, should anyone be interested in linguistic annotation of Swedish texts, there is a research unit in GU with a web tool (and API) for that. The other tools are often for English mainly (sometimes language independent).

You will probably need to run some statistics and analysis in your data. We recommend R, and you can use this quick tutorial to learn the basic statistics in R.
